X & XStudio基础入门

 

需要了解在前面的:

 

· X的语法与C / C++ , Java 极其相似, 对于已经熟练使用这两门语言中的任意一门的程序员来说, 即已经掌握了X。

· X是编译型语言

· X生成的程序是在X虚拟机中运行的.

· X是跨平台的(windows x86 x 64 winrt, linux x86 x64 arm arm64 mips, mac osx x64 , ios, android ), 并且可以使用X外壳直接生成平台所能支持的独立可执行程序.

· X使用了QT作为基础UI 库, QT是一个优秀的跨平台UI库, 可使其良好运行在任意操作系统平台.

· X使用自动的内存管理, 无需担心内存泄露

· X目前未使用jit编译引擎

· X适合用来对项目框架进行快速构建和开发

· X支持快速调用任何平台的C动态库程序接口(API), 仅需声明

· X使用XStudio 来开发app

· XStudio是使用X 开发的, 并且它是开源的.

 

 

 

第一个X程序

下面看一个简单的X程序，它将在控制台窗口打印字符串 Hello World

实例

class HelloWorld {

 /* 第一个X程序 * 它将打印字符串 Hello World */ 

static int main(String []args) { 

_system_.createConsole();

// 创建控制台窗口 

_system_.consoleWrite("Hello X"); 

// 在控制台中打印 Hello World 

return 0;

} 

};

 

下面将逐步介绍如何用XStudio来创建控制台项目：

 

· 打开XStudio，点击菜单[文件] -> [新建] -> [项目]

· 在弹出的 [创建项目] 窗口中, 左侧选择 [项目] -> [xlang],  右侧选择[Xlang 控制台应用]

· 输入一个项目名称, 和工作目录(新创建的项目将被放到这个目录下), 然后点击[创建]按钮。

· 如果没有问题的话, 可以看到项目XStudio 已经加载了创建好的项目, 此时按下键盘上的[F5]键, 编译运行.

 

 

基本语法

编写X程序时，应注意以下几点：

 

· 大小写敏感：X是大小写敏感的，这就意味着标识符Hello与hello是不同的。

· 类名：对于所有的类来说，类名的首字母应该大写。如果类名由若干单词组成，那么每个单词的首字母应该大写，例如 MyFirstXClass 。

· 方法名：所有的方法名都应该以小写字母开头。如果方法名含有若干单词，则后面的每个单词首字母大写。

· 源文件名：源文件名不比和类名相同。当保存文件的时候，你应该使用类名作为文件名保存（切记X是大小写敏感的），文件名的后缀为.xcs 或者 xcsm (与xcs完全相同, 只是为了做感官区分)。（如果文件名和类名不相同不会导致编译错误）。

· 主方法入口：所有的X 程序由类成员函数static int main(String []args) 或者全局函数 int main(String[] args) 方法开始执行 。

· 访问控制: X没有访问控制, 所有对象默认访问权限public

 

 

 

X标识符

 

X所有的组成部分都需要名字。类名、变量名以及方法名都被称为标识符。

 

关于X标识符，有以下几点需要注意：

 

· 所有的标识符都应该以字母（A-Z或者a-z）,或者下划线（_）开始

· 首字符之后可以是字母（A-Z或者a-z）,下划线（_）或数字的任何字符组合

· 关键字不能用作标识符

· 标识符是大小写敏感的

· 合法标识符举例：age、_value、__1_value

· 非法标识符举例：123abc、-salary

X注释

类似于C/C++ Java，X也支持单行以及多行注释。注释中的字符将被编译器忽略。

class HelloWorld { 

/* 这是第一个X程序 *它将打印Hello World * 这是一个多行注释的示例 */ 

static int main(String []args){ 

// 这是单行注释的示例 

/* 这个也是单行注释的示例 */ 

_system_.output("Hello World"); 

} 

};

 

 

 

空行

空白行，或者有注释的行，X编译器都会忽略掉。

 

 

 

继承

在X中，一个类可以由其他类派生。如果你要创建一个类，而且已经存在一个类具有你所需要的属性或方法，那么你可以将新创建的类继承该类。

利用继承的方法，可以重用已存在类的方法和属性，而不用重写这些代码。被继承的类称为超类（super class），派生类称为子类（subclass）。

 

 

 

接口

在X中，接口可理解为对象间相互通信的协议。接口在继承中扮演着很重要的角色。

接口只定义派生要用到的方法，但是方法的具体实现完全取决于派生类。

 

 

 

 

 

远程调试

 

被调试程序以 -xdbn:port 调试模式启动, 应用程序的其他参数在此参数之后

 

例如:

 

server.exe -xdbn:23327 

 

被调试程序以调试模式启动后将在程序代码运行前等待调试端连接.

 

 

 

调试端运行XStudio 后依次点击菜单 [调试] -> [远程调试] , 在弹出的对话框中输入被调试端的IP地址和启动时输入的端口号(上述参数中的23327), 点击[确定]开始调试.
